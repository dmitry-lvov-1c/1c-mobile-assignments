# 📱 Мобильное приложение клиента «Груминг-салон» (1С:Мобильная платформа)

## 📌 Описание проекта и архитектура
Автономное клиентское мобильное приложение, разработанное на «1С:Мобильной платформе» для бесшовного взаимодействия с центральной конфигурацией «Груминг-салон». Приложение автоматизирует процесс удаленного взаимодействия клиента со сферой услуг: позволяет пользователям проходить регистрацию и кастомную авторизацию на устройстве, отслеживать личный кабинет, просматривать справочники доступных мастеров и услуг, а также формировать онлайн-запись на процедуры.

Обмен данными с центральной ERP-системой реализован по REST API с использованием пакетов в формате JSON.

---

## 🚀 Ключевой реализованный функционал

### 1. Архитектура метаданных и кэширование
* **Реализация:** Разработаны локальные мобильные объекты для хранения информации на самом устройстве (справочники мастеров, услуг, пользователей).
* **Бизнес-эффект:** Данные кэшируются в локальной базе смартфона при синхронизации с центральной базой. Приложение работает быстро, минимизирует количество сетевых запросов и экономит мобильный трафик клиента.

### 2. Пользовательский интерфейс и логика авторизации
* **Реализация:** Спроектированы специализированные кастомные формы для управления сессией пользователя: экраны «Зарегистрироваться», «Авторизоваться» и механизм сброса сессии «Выйти».
* **Бизнес-эффект:** Стандартные диалоги платформы заменены на удобный, интуитивно понятный интерфейс клиентского приложения. Доступ к ключевым функциям записи блокируется до прохождения корректной авторизации.

### 3. Механизм оформления онлайн-записи
* **Реализация:** Разработаны документы фиксации заявок и интерактивные формы-мастера для подбора параметров услуги.
* **Бизнес-эффект:** Клиент может самостоятельно выбрать породу питомца, мастера, свободную дату и время, после чего сформированный пакет данных автоматически передается на сервер.

### 4. Интеграция по протоколу HTTP (JSON)
* **Реализация:** Программная логика клиент-серверного взаимодействия вынесена в специализированные общие модули. Мобильный клиент самостоятельно упаковывает данные в формат JSON и обрабатывает входящие ответы сервера.
* **Бизнес-эффект:** Обеспечен двусторонний обмен данными в режиме реального времени. Документы, созданные на мобильном устройстве, мгновенно отправляются через методы POST в центральный HTTP-сервис.

## 📂 Инструкция по развёртыванию и системные требования
1. Создайте пустую мобильную информационную базу в Конфигураторе «1С:Предприятие 8.3».
2. Выберите пункт меню: *Конфигурация ➡️ Загрузить конфигурацию из файлов...* и укажите папку из данного репозитория.
3. Обновите конфигурацию базы данных (клавиша **F7**).
